# Linear Hygiene

Linear reflects reality. Never let tickets drift from actual state.

## States

`Backlog` ‚Üí `Todo` ‚Üí `In Progress` ‚Üí `In Review` ‚Üí `Done`

## End-of-session checklist

Run this at the end of every session, without exception:

- Move any in-progress tickets to the correct state.
- Add a progress comment to every ticket worked on during the session.
- Create new tickets for any work discovered during the session.
- Link every ticket to its relevant commit(s) before closing.
- Never end a session with Linear stale or inaccurate.

## Priority

- **P1 Urgent** ‚Äî production down, security breach, data loss, or users blocked from core functionality right now. Use sparingly.
- **P2 High** ‚Äî blocker for other planned work, or a significant user-affecting bug. Default for schema and infrastructure work.
- **P3 Medium** ‚Äî standard feature work, improvements, non-blocking bugs. Default for most feature tickets.
- **P4 Low** ‚Äî nice-to-haves, polish, backlog items with no near-term dependency.
